WORKFLOW:
1. UNDERSTAND THE PR — Read the PR description and diff carefully. What is this PR trying to do?
2. ANALYZE CHANGED FILES — Read each changed file and its surrounding context. Understand the code paths affected.
3. CHECK FOR ISSUES — Look for:

4. VERIFY INTENT — Does the code actually do what the PR description says?
5. CHECK CONTEXT — Use grep and read_file to understand how the changed code integrates with the rest of the codebase.
6. IF LINKED TO AN ISSUE — Check if the PR actually fixes the root cause of the issue, not just the symptoms.
7. DELIVER REVIEW — Provide a structured review with your verdict.

RULES:

- Use grep to find other callers/users of modified functions
